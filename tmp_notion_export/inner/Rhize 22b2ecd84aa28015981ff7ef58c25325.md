# Rhize

CPU Speed (GHz): 3.2
Highly Available: Yes
Memory (GB) Per Node: 32
Nodes: 3
PV IOPs: 5000
PV Throughput (mbps): 500
Persisted Volumes: 20
Supported Product Versions: v4.0.0 (https://www.notion.so/v4-0-0-19e2ecd84aa280c69d53ce86239a219b?pvs=21)
Total Disk Space (TB): 3
vCPU per Node: 16

**Only Kubernetes is supported in Production Environments.**

## Components

- Rhize BAAS in HA which includes
    - 3x Alpha Pods
        - Has Stateful PV, want high IOPS (5000) & Throughput (500MBps) and 750GB
    - 3x Zero Pods
        - Has Stateful PV, want high IOPS (5000) & Throughput (500MBps) and 300GB
    - **Avoid NFS or SMB Filesystems - These are known to lead to corruption**
        - This includes Azure Container App
- ISA-95 Service
    - 1x Replicas of ISA-95
        - No HA Option Available ( we are working on this though)
        - No Storage
- Workflow
    - 3x Replicas of Workflow
        - No Storage
- Redpanda in HA which includes
    - 3x Redpanda Pods
        - Has Stateful PV, want high IOPS and 100GB
        - **Dont’ use NFS or SMB Filesystems - these won’t work - Redpanda will fail to even startup**
            - This includes Files Shares in Azure Container Apps (ACA)
    - 1x Console
- Grafana HA
    - 3x Replicas of Grafana
        - Host storage, AWS EFS or event S3 Storage class is acceptable, 20GB
- 2x Libre Audit
    - 2x Libre Audit Pods
        - No Storage
    - 2x Postgres
        - Has Stateful PV, 250GB
- Libre Admin
    - 3x Replicas of Libre Admin
        - No Storage
- Keycloak in HA
    - 2x Keycloak Pods
        - Has small volume for theme <1GB
    - 2x Postgres HA Pods
        - Has Stateful PV, 250GB
- QuestDB
    - 1x QuestDB Pod
        - Has Stateful PV, want high IOPS (7000) 500MBps throughput  and 250GB
            - Filesystem:
                - **Can’t Use SMB / NFS,  (this includes Azure Container Apps)**
                - needs to be either **ext4** is commonly used
                Ok to use **ZFS** with **lz4** compression being a better option for advanced setups.
        - Minimum - 4vCPU 8GB
- 2
    - Has Stateful PV, want high IOPS and 50GB
    - **Dont’ use NFS or SMB Filesystems - these won’t work - Restate will initially work but it will eventually lead to corruption**
        - This includes Files Shares in Azure Container Apps (ACA)
- Solace
    - ???x Solace Pods
        - Has Stateful PV, want high IOPS and 100GB
- AppSmith
    - Unsure on what HA setup is
        - Has Stateful PV, 50GB can be slower (1000 IOPS is ok)

General recommendations for running rhize is a K8s cluster with 3 nodes of 64GB with 16vCPUs with a total storage of up to 2TB of 5000 IOPS/500MBps volumes.

| # of nodes | CPU per Nodes | CPU Performance | Memory | Network Speed | Persistent Volume Capacity per Node | Persistent Volume IOPS | Persistent Volume Throughput |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 3 | 16 | +3.2GHz | 65GB | 10GB/p | 2TB | 5000 IOPS | 500 MBps |
